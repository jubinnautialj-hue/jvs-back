# 接口文档

Jvs框架集成了Knife4j在线文档

### 1、spring boot demo项目中在pom.xml中添加
~~~
 <dependency>
	 <groupId>cn.bctools</groupId>
	 <artifactId>jvs-starter-knife4j</artifactId>
 </dependency>
~~~
或添加  `jvs-start-web`中依赖了`jvs-starter-knife4j`
~~~
 <dependency>
	 <groupId>cn.bctools</groupId>
	 <artifactId>jvs-starter-web</artifactId>
 </dependency>
~~~
### 2、在bootstart.yml中添加服务说明
~~~
swagger:
  title: "服务名称"
  description: "服务描述"
~~~
### 3、访问本地文档`http://localhost:{port}/swagger-ui/#/`

---
Knife4j是一款基于Swagger的API文档在线生成与管理工具，它提供了丰富的功能和友好的界面，方便开发人员快速创建和查阅API文档。

下面是Knife4j的一些特点和功能：

1. 自动生成API文档：Knife4j可以自动扫描项目中的API接口，根据接口注解生成对应的API文档，并提供可视化的API界面。

2. 丰富的文档展示：Knife4j支持在文档中显示接口信息、请求参数、响应结果以及错误码等详细信息。还可以展示接口的调试功能，方便开发人员进行接口测试。

3. 支持Markdown编辑：Knife4j支持使用Markdown语法编辑API文档中的描述信息，使文档更加易读和易维护。

4. 参数校验和测试：Knife4j提供了数据校验功能，可以对请求参数进行校验，确保数据的合法性。同时，还支持在线测试接口，可以直接在API文档页面进行接口调用和测试。

5. 权限管理：Knife4j支持对API文档进行权限管理，可以限制不同用户对不同接口的访问权限，保护敏感接口的安全性。

6. 模块化管理：Knife4j支持将API接口按照模块进行分类管理，方便开发人员对不同的接口进行组织、查找和管理。

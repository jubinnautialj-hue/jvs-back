### 一 源代码添加逻辑引擎节点

这里只需要后端添加即可。前端会根据添加的后端组件自动渲染。

~~~
@Service
@AllArgsConstructor
@Rule(value = "生成唯一ID",
		//节点分组
        group = RuleGroup.工具插件,
		//节点解释
        explain = "生成唯一ID",
		//是否支持测试
        test = true,
		//节点返回字段类型
        returnType = ClassType.文本,
		//测试结果显示样式
        testShowEnum = TestShowEnum.TEXT
)
public class IdNumberServiceImpl implements BaseCustomFunctionInterface {

    /**
     * 执行节点
     * @param o  执行参数对象
     * @param params  全局参数对象
     * @return
     */
    @Override
    public Object execute(Object o, Map params) {
        return IdWorker.get32UUID();
    }
}
~~~

### 三 使用注解添加组件

* 添加pom.xml 包
```
 <!--逻辑引擎集成注解-->
        <dependency>
            <groupId>cn.bctools</groupId>
            <artifactId>jvs-function-annotation</artifactId>
            <version>{version}</version>
        </dependency>
```
* 添加 controller 方法注解
```
    @Rule(value = "服务注册示例", group = RuleGroup.服务插件, explain = "服务节点注册逻辑的示例,包含请求头，url 参数接收值")
    @ApiOperation(value = "服务注册示例")
    @PostMapping("/rule/test/{path}")
    public R<Object> ruleTest(@RequestHeader(value = "header", required = false) String header, @PathVariable("path") String path, @RequestBody Test test) {
            // 业务逻辑
            return R.ok(test);
    }
```

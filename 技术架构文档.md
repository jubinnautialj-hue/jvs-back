# JVS 微服务框架技术架构文档

## 一、项目概述

JVS 是一套基于 Spring Cloud Alibaba 实现的微服务开源框架，支持低代码开发、逻辑引擎、规则引擎、数据智仓、企业文档等功能。

### 1.1 核心特性

- **低代码开发**：支持可视化表单设计、流程设计、逻辑编排
- **逻辑引擎**：提供灵活的规则引擎和公式引擎
- **数据智仓**：集成 Apache Doris 提供数据处理和分析能力
- **企业文档**：支持富文本、Excel、脑图等在线编辑预览
- **微服务架构**：基于 Spring Cloud Alibaba 的分布式架构

### 1.2 版本信息

| 组件 | 版本 |
|------|------|
| Spring Boot | 2.7.12 |
| Spring Cloud | 2021.0.8 |
| Spring Cloud Alibaba | 2021.0.5.0 |
| JDK | 1.8 |
| MyBatis Plus | 3.5.3.1 |
| Hutool | 5.8.24 |

### 5.2 网关服务 (jvs-gateway)

**核心功能：**

1. **路由转发**
   - 动态路由配置（基于Nacos）
   - 服务发现自动路由

2. **认证鉴权**
   - OAuth2 Token校验
   - 资源权限校验
   - 白名单配置

3. **安全防护**
   - XSS攻击拦截
   - 请求响应加密
   - 接口版本控制

4. **监控日志**
   - 请求时间记录
   - 业务错误码转换
   - Swagger文档聚合

**核心过滤器：**

| 过滤器 | 功能 |
|--------|------|
| AuthGlobalFilter | 权限判断与认证 |
| GlobalRequestBodyDecodeFilter | 请求响应加解密 |
| GrayReactiveLoadBalancerClientFilter | 版本路由/灰度 |
| RequestTimeFilter | 请求耗时记录 |
| XssFilter | XSS攻击防护 |

基于Spring Security OAuth2实现的认证服务：

**核心功能：**
- 用户名密码登录

- 多租户支持

**核心注解：**
- `@EnableJvsMgrResourceServer`：启用资源服务
- `@EnableJvsFeignClients`：启用Feign客户端

### 5.4 逻辑引擎

**核心组件：**

1. **公式引擎** (`jvs-starter-function`)
   - 表达式解析与执行
   - 自定义函数注册
   - 公式场景管理

2. **逻辑编排** (`jvs-function`)
   - 基础节点（条件、循环、变量）
   - 数据节点（数据库操作）
   - 服务节点（API调用）
   - 系统节点（用户信息、时间）

**核心注解：**
- `@Rule`：标记逻辑节点
- `@ParameterValue`：参数注解

## 六、服务间通信

### 6.1 Feign调用

**使用方式：**

1. 在启动类添加注解：
```java
@EnableJvsFeignClients
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

2. 定义API接口（在-api模块）：
```java
@FeignClient(value = "jvs-auth-mgr", contextId = "UserApi")
public interface UserApi {
    @PostMapping("/mgr/user/list")
    R<List<UserDTO>> listUsers(@RequestBody QueryDTO query);
}
```

3. 服务消费者注入使用：
```java
@Autowired
private UserApi userApi;
```

## 七、多租户架构

### 7.1 租户隔离方案

**基于MyBatis Plus的多租户插件实现：**

```
┌─────────────────────────────────────────────┐
│              请求进入                          │
│  Header: tenant_id (可选，默认登录用户)         │
└───────────────────┬─────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│         TenantInterceptor (拦截器)            │
│         解析Header中的租户ID                    │
└───────────────────┬─────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│      MyBatis Plus TenantLineInnerInterceptor │
│      自动在SQL中添加 tenant_id = ? 条件         │
└───────────────────┬─────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│              数据库执行                        │
│         自动过滤非当前租户数据                   │
└─────────────────────────────────────────────┘
```

### 7.2 核心实现

**配置类位置：** `jvs-starter-database` 模块

**相关注解：**
- `@TenantLine`：标记需要租户隔离的表
- `@IgnoreTenant`：忽略租户隔离（管理员使用）

## 八、安全架构

### 8.1 认证流程

```
┌─────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  客户端  │────▶│  网关    │────▶│  认证服务 │────▶│  数据库   │
└─────────┘     └──────────┘     └──────────┘     └──────────┘
     │                │                │                │
     │  1. 登录请求    │                │                │
     │───────────────▶│                │                │
     │                │  2. 转发到认证  │                │
     │                │───────────────▶│                │
     │                │                │ 3. 验证用户密码  │
     │                │                │───────────────▶│
     │                │                │                │
     │                │                │◀───────────────│
     │                │  4. 返回Token   │                │
     │                │◀───────────────│                │
     │  5. 返回Token   │                │                │
     │◀───────────────│                │                │
     │                │                │                │
     │  6. 携带Token请求│                │                │
     │───────────────▶│                │                │
     │                │  7. 校验Token   │                │
     │                │───────────────▶│                │
     │                │                │ 8. 验证Token有效性│
     │                │                │───────────────▶│
     │                │                │                │
     │                │                │◀───────────────│
     │                │  9. 转发到业务服务│                │
     │                │───────────────▶│                │
     │                │                │                │
```

### 8.2 主要安全特性

| 特性 | 说明 |
|------|------|
| OAuth2.0 | 标准授权协议，支持多种授权模式 |
| JWT Token | 无状态认证，Token中包含用户信息 |
| 接口加密 | /mgr/ 开头的接口请求响应加密 |
| XSS防护 | 网关层XSS攻击拦截 |
| 权限控制 | 基于角色的权限控制(RBAC) |
| 数据权限 | 行级数据权限控制 |

### 8.3 核心工具类

- `UserCurrentUtils`：获取当前登录用户
- `AuthorityManagementUtils`：获取用户权限/角色/岗位

## 十一、扩展点

### 11.1 可扩展模块

| 模块 | 扩展方式 |
|------|----------|
| 逻辑节点 | 继承`BaseCustomFunctionInterface`，添加`@Rule`注解 |
| 表单组件 | 实现`IDataFieldHandler`接口 |
| 数据源 | 实现`DataSourceExecuteInterface`接口 |
| 图表类型 | 继承`BaseListChartBo`类 |
| 工作流节点 | 实现工作流节点处理器 |

### 11.2 配置化开发

项目核心设计理念是**配置化开发**，通过以下方式实现：

1. **表单配置化**：无需编码，通过拖拽组件生成表单
2. **列表配置化**：通过配置定义列表字段、查询条件、操作按钮
3. **流程配置化**：可视化设计工作流，定义审批节点、流转条件
4. **数据配置化**：通过数据集设计器定义数据处理流程
5. **逻辑配置化**：通过规则引擎可视化编排业务逻辑

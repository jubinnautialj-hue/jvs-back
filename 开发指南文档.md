# JVS 微服务框架开发指南

## 一、快速开始

### 1.1 环境准备

#### 1.1.1 必需软件

| 软件 | 版本 | 说明 |
|------|------|------|
| JDK | 1.8+ | Java开发环境 |
| Maven | 3.8+ | 项目构建工具 |
| IDE | IntelliJ IDEA / Eclipse | 开发IDE（推荐IDEA） |
| Git | - | 版本控制 |

#### 1.1.2 可选软件（本地开发）

| 软件 | 版本 | 说明 |
|------|------|------|
| Nacos | 2.x | 服务注册与配置中心 |
| MySQL | 8.0+ | 关系数据库 |
| Redis | 6.0+ | 缓存服务 |
| RabbitMQ | 3.8+ | 消息队列 |
| Apache Doris | 1.2+ | 数据仓库（BI模块需要） |

### 1.2 项目导入

#### 1.2.1 Maven配置

在 `settings.xml` 中配置镜像（可选）：

```xml
<mirrors>
    <mirror>
        <id>aliyun</id>
        <mirrorOf>central</mirrorOf>
        <name>Aliyun Maven Mirror</name>
        <url>https://maven.aliyun.com/repository/central</url>
    </mirror>
</mirrors>
```

#### 1.2.2 IDEA导入步骤

1. 打开 IntelliJ IDEA
2. File → Open → 选择项目根目录 `jvs-back`
3. 等待 Maven 依赖下载完成
4. 确保 Project SDK 设置为 JDK 1.8
5. Maven Profiles 选择需要的模块

### 1.3 项目结构理解

```
jvs-back/
```

## 二、配置说明

### 2.1 配置文件结构

每个服务的配置分为两部分：

1. **本地配置** (`bootstrap.yml`) - 基础配置、Nacos连接
2. **远程配置** (Nacos) - 业务配置、数据库连接等

### 2.2 Bootstrap.yml 配置示例

```yaml
server:
  port: 10000                    # 服务端口
version: @project.version@
spring:
  application:
    name: @project.artifactId@   # 服务名称
  main.allow-bean-definition-overriding: true
  cloud:
    nacos:
      discovery:
        server-addr: 127.0.0.1:8848  # Nacos地址
        group: jvs
        namespace: ${spring.cloud.nacos.discovery.group}
        username: nacos              # Nacos用户名（如需要）
        password: nacos              # Nacos密码（如需要）
      config:
        server-addr: ${spring.cloud.nacos.discovery.server-addr}
        group: ${spring.cloud.nacos.discovery.group}
        namespace: ${spring.cloud.nacos.discovery.namespace}
    inetutils:
      preferred-networks: 10.*    # 首选网卡网段
  config:
    import:
      - optional:nacos:application.yml
      - optional:nacos:${spring.application.name}.yml
```

## 三、服务启动顺序

### 3.1 基础设施启动

按以下顺序启动基础设施：

1. **Nacos** - 服务注册与配置中心
2. **MySQL** - 关系数据库
3. **Redis** - 缓存服务
4. **RabbitMQ** - 消息队列（可选）
5. **Apache Doris** - 数据仓库（BI模块需要）

### 3.2 微服务启动顺序

```
第1步: jvs-gateway          (网关服务 - 端口: 10000)
      ↓
第2步: jvs-auth             (认证服务 - 端口: 动态)
      ↓
第3步: jvs-auth-mgr         (管理服务 - 端口: 动态)
      ↓
第4步: 按需启动业务服务
      ├── jvs-design-mgr    (低代码设计)
      ├── jvs-data-factory-mgr (数据工厂)
      ├── jvs-chart-mgr     (图表服务)
      ├── jvs-data-report-mgr (报表服务)
      ├── jvs-data-screen-mgr (大屏服务)
      ├── document-mgr      (文档管理)
      ├── message-push-mgr  (消息推送)
      └── jvs-xxl-job-admin (任务调度)
```

### 3.3 启动验证

1. 访问 Nacos 控制台：`http://127.0.0.1:8848/nacos`
2. 查看服务列表，确认服务已注册
3. 访问网关：`http://127.0.0.1:10000`
4. 访问 Swagger 文档：`http://127.0.0.1:10000/doc.html`

### 4.3 统一响应格式

项目使用统一的响应对象 `R<T>`：

#### 4.3.1 响应结构

```java
public class R<T> {
    private int code;           // 状态码
    private String msg;         // 消息
    private T data;             // 数据
    private Long time;          // 时间戳
}
```

#### 4.3.2 状态码定义

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权/登录失效 |
| 403 | 权限不足 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

#### 4.3.3 使用示例

```java
@RestController
@RequestMapping("/mgr/user")
public class UserController {
    
    @GetMapping("/{id}")
    public R<UserDTO> getById(@PathVariable String id) {
        UserDTO user = userService.getById(id);
        return R.data(user);
    }
    
    @PostMapping
    public R<Boolean> save(@RequestBody UserDTO user) {
        boolean result = userService.save(user);
        return R.success(result);
    }
    
    @DeleteMapping("/{id}")
    public R<Boolean> delete(@PathVariable String id) {
        userService.removeById(id);
        return R.success();
    }
}
```

### 4.5 Controller 规范

#### 4.5.1 标准Controller结构

```java
@RestController
@RequestMapping("/mgr/user")
@Api(tags = "用户管理")
@Slf4j
public class UserController {
    
    @Autowired
    private UserService userService;

    public R<IPage<UserDTO>> page(

    @ApiOperation("保存用户")

        boolean result = userService.saveOrUpdate(user);
        return R.success(result);
    }
    
    @DeleteMapping("/{id}")
    @ApiOperation("删除用户")
```

3. **POST请求的复杂参数必须使用 `@RequestBody`**

## 六、数据访问层

### 6.1 MyBatis Plus 使用

#### 6.1.1 Mapper接口

```java
public interface UserMapper extends BaseMapper<SysUser> {
    
    @Select("SELECT * FROM sys_user WHERE user_name = #{userName}")
    SysUser selectByUserName(@Param("userName") String userName);
    
    List<UserDTO> selectUserWithRole(@Param("query") UserQueryDTO query);
}
```

#### 6.1.2 Service层

```java
public interface UserService extends IService<SysUser> {
    
    UserDTO getDetailById(String id);
    
    IPage<UserDTO> pageUser(Page<UserDTO> page, UserQueryDTO query);
}
```

```java
@Service
public class UserServiceImpl extends ServiceImpl<UserMapper, SysUser> implements UserService {
    
    @Override
    public UserDTO getDetailById(String id) {
        SysUser user = this.getById(id);
        return BeanUtils.copyProperties(user, UserDTO.class);
    }
    
    @Override
    public IPage<UserDTO> pageUser(Page<UserDTO> page, UserQueryDTO query) {
        LambdaQueryWrapper<SysUser> wrapper = Wrappers.lambdaQuery();
        
        if (StrUtil.isNotBlank(query.getUserName())) {
            wrapper.like(SysUser::getUserName, query.getUserName());
        }
        if (query.getStatus() != null) {
            wrapper.eq(SysUser::getStatus, query.getStatus());
        }
        
        wrapper.orderByDesc(SysUser::getCreateTime);
        
        Page<SysUser> userPage = new Page<>(page.getCurrent(), page.getSize());
        this.page(userPage, wrapper);
        
        // 转换DTO
        List<UserDTO> records = userPage.getRecords().stream()
            .map(user -> BeanUtils.copyProperties(user, UserDTO.class))
            .collect(Collectors.toList());
        
        page.setRecords(records);
        page.setTotal(userPage.getTotal());
        
        return page;
    }
}
```

### 6.2 多租户使用

#### 6.2.1 租户自动隔离

配置多租户后，查询会自动添加租户条件：

```java
// 原SQL: SELECT * FROM sys_user WHERE user_name = 'admin'
// 实际执行: SELECT * FROM sys_user WHERE user_name = 'admin' AND tenant_id = 'xxx'
```

#### 6.2.2 忽略租户隔离

在需要忽略租户的方法上添加 `@IgnoreTenant` 注解：

```java
@IgnoreTenant
@GetMapping("/all")
public R<List<UserDTO>> listAll() {
    // 此查询不会添加租户条件
    List<SysUser> users = userService.list();
    return R.data(convert(users));
}
```

### 6.3 事务管理

#### 6.3.1 声明式事务

```java
@Service
public class UserServiceImpl implements UserService {
    
    @Transactional(rollbackFor = Exception.class)
    public void saveUserWithRole(UserDTO user, List<String> roleIds) {
        // 1. 保存用户
        this.save(user);
        
        // 2. 保存用户角色关系
        userRoleService.saveBatch(buildRelations(user.getId(), roleIds));
        
        // 3. 如果抛出异常，事务回滚
        if (user.getUserName().equals("error")) {
            throw new BusinessException("测试回滚");
        }
    }
}
```

#### 6.3.2 分布式事务（Seata）

项目集成了 Seata，需要时可使用 `@GlobalTransactional`：

```java
@GlobalTransactional
@Service
public class OrderService {
    
    @Autowired
    private StockApi stockApi;
    
    @Autowired
    private AccountApi accountApi;
    
    @Transactional(rollbackFor = Exception.class)
    public void createOrder(OrderDTO order) {
        // 1. 创建订单
        this.save(order);
        
        // 2. 扣减库存（远程调用）
        R<Boolean> stockResult = stockApi.deduct(order.getProductId(), order.getQty());
        if (!stockResult.isSuccess()) {
            throw new BusinessException("库存不足");
        }
        
        // 3. 扣减余额（远程调用）
        R<Boolean> accountResult = accountApi.deduct(order.getUserId(), order.getAmount());
        if (!accountResult.isSuccess()) {
            throw new BusinessException("余额不足");
        }
    }
}
```

```java
@Component
@Slf4j
public class DemoJobHandler {
    
    @XxlJob("demoJobHandler")
    public void execute() throws Exception {
        log.info("开始执行定时任务...");
        
        // 获取任务参数
        String param = XxlJobHelper.getJobParam();
        log.info("任务参数: {}", param);
        
        try {
            // 业务逻辑
            this.doBusiness();
            
            // 任务成功
            XxlJobHelper.handleSuccess();
        } catch (Exception e) {
            log.error("任务执行失败", e);
            XxlJobHelper.handleFail("执行失败: " + e.getMessage());
            throw e;
        }
    }
    
    private void doBusiness() {
        // 实际业务逻辑
    }
}
```

#### 9.1.3 XXL-Job Admin 配置

在 XXL-Job 控制台添加任务：

1. 进入 `任务管理` → `新增任务`
2. 配置：
   - 执行器：选择对应的服务执行器
   - 任务描述：任务说明
   - Cron：定时表达式（如 `0 0 2 * * ?` 表示每天凌晨2点）
   - JobHandler：填写 `@XxlJob` 注解中的值（如 `demoJobHandler`）
   - 任务参数：可选参数

```java
@Component
@RabbitListener(queues = "user.create.queue")
@Slf4j
public class UserCreateListener {
    
    @RabbitHandler
    public void process(String message, Channel channel, 
                        @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
        try {
            log.info("收到消息: {}", message);
            
            // 解析消息
            UserDTO user = JSON.parseObject(message, UserDTO.class);
            
            // 业务处理
            this.handleUserCreate(user);
            
            // 手动确认
            channel.basicAck(deliveryTag, false);
            
        } catch (Exception e) {
            log.error("消息处理失败", e);
            try {
                // 拒绝消息，重新入队
                channel.basicNack(deliveryTag, false, true);
            } catch (IOException ex) {
                log.error("消息拒绝失败", ex);
            }
        }
    }
    
    private void handleUserCreate(UserDTO user) {
        // 业务逻辑
    }
}
```

```java
@GetMapping("/download/{fileId}")
public void download(@PathVariable String fileId, HttpServletResponse response) {
    // 获取文件信息
    FileInfo fileInfo = fileTemplate.getFileInfo(fileId);
    
    // 设置响应头
    response.setContentType("application/octet-stream");
    response.setHeader("Content-Disposition", 
        "attachment;filename=" + URLEncoder.encode(fileInfo.getFileName(), "UTF-8"));
    
    // 下载文件

         OutputStream outputStream = response.getOutputStream()) {
        IOUtils.copy(inputStream, outputStream);
    }
}
```

## 十四、开发常见问题

### 14.1 Maven依赖下载失败

**问题**：无法下载依赖或下载缓慢

**解决**：
1. 检查网络连接
2. 配置阿里云镜像（见1.2.1节）
3. 清理本地仓库后重新下载：
```bash
mvn clean install -U
```

### 14.2 服务无法注册到Nacos

**问题**：服务启动后Nacos服务列表为空

**解决**：
1. 检查Nacos是否启动
2. 检查 `bootstrap.yml` 中Nacos地址配置
3. 检查防火墙/网络是否可访问
4. 检查Nacos用户名密码配置

### 14.3 数据库连接失败

**问题**：服务启动时报数据库连接错误

**解决**：
1. 检查MySQL是否启动
2. 检查Nacos配置中心的数据库配置
3. 检查数据库用户权限
4. 检查时区配置（serverTimezone=GMT%2B8）

### 14.4 多租户数据隔离问题

**问题**：查询不到数据或查询到其他租户数据

**解决**：
1. 确认请求头包含正确的 `tenant_id`
2. 检查实体类是否正确配置租户字段
3. 使用 `@IgnoreTenant` 时注意影响范围

### 14.5 Feign调用失败

**问题**：服务间调用返回错误

**解决**：
1. 检查目标服务是否已启动
2. 检查Feign接口路径是否正确
3. 检查是否返回 `R<T>` 对象
4. 检查参数注解（@PathVariable必须指定名称）
